# Architecture

Desktop app on GPUI, monorepo with Cargo workspace.

## Structure

```
gpui_test/
├── Cargo.toml                # Конфигурация воркспейса
├── crates/
│   ├── audio_common/         # Общие типы данных (shared)
│   ├── audio_decoder/        # Чтение файлов (Symphonia)
│   ├── audio_dsp/            # Обработка звука (Громкость, Эффекты)
│   ├── audio_output/         # Взаимодействие с железом (CPAL/ALSA)
│   └── audio_engine/         # Оркестратор (Управление потоками и логикой)
└── tests/                    # Интеграционные тесты
```

Подробное описание крейтов
1. audio_common
Самый маленький и важный крейт. Он не содержит логики, только определения, о которых договорились все остальные модули.
enum AudioBuffer: Контейнер для сэмплов. Пока содержит только F32(Vec<f32>). Позволяет избежать переписывания кода в будущем при добавлении I32 или DSD.
struct StreamParams: Метаданные потока: sample_rate, channels, bit_depth.
enum AudioError: Единый стандарт ошибок для всего проекта.
2. audio_decoder
Интерфейс между музыкальным файлом и сырыми данными.
Технологии: symphonia.
Интерфейс (Trait): AudioSource.
Внутри:
Логика открытия файла и автоматического определения формата (MP3, FLAC и т.д.).
Конвертер: берет данные из внутреннего буфера Symphonia и упаковывает их в наш AudioBuffer::F32.
Нюанс: Должен корректно обрабатывать "хвосты" файлов (padding), чтобы обеспечить Gapless (воспроизведение без щелчков между треками).
3. audio_dsp
Математическая лаборатория. Только вычисления, никакой работы с файлами или сетью.
Интерфейс (Trait): DspProcessor.
Модули:
Volume: Узел управления громкостью. Имеет флаг bypass. Если включен Bit-Perfect, узел просто пробрасывает данные без умножения.
Converter: (на будущее) для пересчета типов данных.
Зачем отдельно: Здесь ты сможешь писать тесты типа: "Если подать сигнал амплитудой 1.0 и выставить громкость 0.5, на выходе должно быть строго 0.5".
4. audio_output
Слой абстракции над звуковой картой.
Технологии: cpal (основной), alsa (опционально для Linux Bit-Perfect).
Интерфейс (Trait): AudioSink.
Логика:
Умеет перечислять доступные устройства (ЦАПы).
Умеет открывать Exclusive Mode (запрашивает у ОС прямой доступ).
Умеет открывать Shared Mode (подстраивается под системный микшер).
Регулирует Hardware Volume (UAC2), если устройство это поддерживает.
5. audio_engine (Core)
«Мозг» системы. Он связывает все крейты в работающий конвейер.
Структура:
Controller: Принимает команды от UI (Play/Pause/Volume) через каналы (mpsc).
Worker: Высокоприоритетный поток, в котором крутится бесконечный цикл:
Взять данные из Decoder.
Прогнать через DSP (если нужно).
Отправить в Output.
RingBuffer: Использует ringbuf для передачи данных между потоком декодирования и потоком вывода (чтобы звук не заикался при нагрузке на диск).
Как это будет работать (Сценарий: Включение Bit-Perfect)
UI посылает команду в audio_engine: SetBitPerfect(true).
audio_engine получает параметры текущего файла из audio_decoder (например, 24-bit / 192kHz).
audio_engine дает команду audio_output: "Пересоздай поток. Мне нужен Exclusive доступ под 192kHz".
audio_engine дает команду audio_dsp: "Включи Bypass для громкости".
Результат: Данные из файла без изменений летят в ЦАП. Ползунок громкости в плеере блокируется (или переключается на Hardware команды).
С чего начать разработку (Порядок действий)
Настройка Workspace: Создать Cargo.toml в корне и пустые папки крейтов.
audio_common: Описать AudioBuffer и StreamParams. Это займет 5 минут, но позволит остальным крейтам "видеть" друг друга.
audio_decoder: Реализовать проигрывание одного FLAC файла в "никуда" (просто вывод сэмплов в консоль). Это подтвердит, что Symphonia настроена верно.
audio_output: Написать минимальный код на cpal, который просто издает "писк" (синусоиду). Это подтвердит работу с аудио-картой.
Соединение: В audio_engine соединить декодер и вывод.

## Build

```bash
cargo build
cargo run -p gpui-test
```

## Conventions

- Crates: lowercase with underscores
- `publish = false` for internal crates
- Platform-specific code via `cfg` attributes
- Version and edition from workspace
